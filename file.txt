

Create a Role with Required Permissions: Use the following YAML to define a Role with permissions to list, watch, and create pods in the new-jenkins3 namespace:

apiVersion: rbac.authorization.k8s.io/v1
kind: Role
metadata:
  namespace: new-jenkins3
  name: jenkins-role
rules:
- apiGroups: [""]
  resources: ["pods", "pods/log"]
  verbs: ["get", "list", "watch"]
- apiGroups: ["apps"]
  resources: ["deployments"]
  verbs: ["get", "list", "watch"]

Bind the Role to the Jenkins Service Account: Create a RoleBinding to assign the role to the Jenkins service account:

apiVersion: rbac.authorization.k8s.io/v1
kind: RoleBinding
metadata:
  name: jenkins-rolebinding
  namespace: new-jenkins3
subjects:
- kind: ServiceAccount
  name: jenkins
  namespace: new-jenkins3
roleRef:
  kind: Role
  name: jenkins-role
  apiGroup: rbac.authorization.k8s.io

Apply the Configuration: Apply both the Role and RoleBinding using:

kubectl apply -f role.yaml
kubectl apply -f rolebinding.yaml
